    Vectors & Embeddings

    An embedding is a numerical representation of text. Words, phrases, or documents are mapped to high-dimensional vectors such that items with similar meaning are located near each other in the vector space.

    Applications

    • Semantic search → retrieving documents based on meaning rather than keywords
    • Clustering and classification → grouping related content automatically
    • Retrieval-augmented generation (RAG) → supplying relevant documents to LLMs during generation
    • Evaluation → comparing model outputs against references using similarity

    Example: Cosine Similarity

    The following code shows how to measure similarity between two vectors using cosine similarity.
    It measures the angle between two vectors — smaller angle = higher similarity.

    def cosine(a, b):
    import math
    # Compute dot product of the two vectors
    dot = sum(x*y for x, y in zip(a, b))

    # Compute magnitude (length) of each vector
    na = math.sqrt(sum(x*x for x in a))
    nb = math.sqrt(sum(x*x for x in b))

    # Divide dot product by product of magnitudes
    return dot / (na * nb + 1e-12)  # small constant avoids division by zero

# Example usage
v1 = [1, 0, 1]
v2 = [0.9, 0.1, 1.1]
print(round(cosine(v1, v2), 4))  # ~0.98 → very similar

    Explanation:

    Dot product → measures overlap between the vectors.

    Magnitudes → normalize for vector length (longer sentences ≠ higher score).

    Cosine similarity → dot / (lengths). Range:

    • 1.0 → identical direction (very similar meaning).

    • 0.0 → orthogonal (no relation).

    • -1.0 → opposite meaning (rare in embeddings).

    Why this matters

    Embeddings allow semantic tasks like:

    • “Find me texts similar to this one.”

    • “Cluster related documents automatically.”

    • “Evaluate if two answers mean the same thing.”
