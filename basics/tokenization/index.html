    Tokenization

    Language models do not process raw text directly. Instead, they operate on discrete units called tokens.
    A token may represent a whole word, a subword, or even a single character depending on the tokenizer.
    Understanding tokenization is essential for controlling cost, managing context limits, and designing effective prompts.

    Why Tokenization Matters

    • Cost & efficiency → most APIs bill per token (both input + output).
    • Context limits → each model has a maximum context window (e.g., 4k, 16k, 128k tokens).
    • Output control → concise prompts = fewer tokens, lower cost, and more predictable outputs.

    Example: Approximate Token Counting

    A rough rule of thumb: 1 token ≈ 4 characters of English text.
    This simple function provides a quick estimate:

    def rough_token_count(text: str) -> int:
    """
    Quick, language-dependent approximation:
    ~1 token per 4 characters of English text.
    Use model-specific tokenizers for production accuracy.
    """
    return max(1, len(text) // 4)

# Example
print(rough_token_count("Hello, world!"))  # ~3

    Explanation

    1) Divide string length by 4 → gives an approximate token count.

    2) max(1, …) ensures even short strings return ≥1 token.

    3) Useful for early estimation, but not reliable for non-English text, emojis, or special characters.

    Accurate Token Counting (Model-Specific)

    For precise counts, always use the tokenizer that matches your model.

    • OpenAI GPT models → tiktoken

    • Many open models (e.g., LLaMA, T5) → SentencePiece

    • General concept → Byte Pair Encoding (BPE)

    # Exact token counting with OpenAI's GPT tokenizer
# Install with: pip install tiktoken
import tiktoken

enc = tiktoken.get_encoding("cl100k_base")  # encoding for GPT-4
text = "Hello, world!"
tokens = enc.encode(text)

print(tokens)      # [9906, 11, 1917, 0]
print(len(tokens)) # exact token count (4 here)

    Explanation

    1) get_encoding("cl100k_base") loads the tokenizer used by GPT-4.

    2) encode(text) converts text → list of token IDs.

    3) len(tokens) gives the exact token count.

    4) Results vary by model → always use the tokenizer for the model you plan to run.

    Practical Guidelines

    • Always budget prompts in tokens, not characters.

    • Place the instruction first and keep it concise.

    • Specify output format (e.g., JSON, bullet points) to compress meaning into fewer tokens.

    • For long docs, chunk into 300–500 tokens with overlap, then re-rank to feed only the most relevant chunks.

    • Limit max_tokens in outputs to control cost and latency.

    Fast sanity checks

    • Use the model vendor’s online tokenizer tool for quick checks.
    • Keep a utility function in your repo to measure token counts before sending requests
